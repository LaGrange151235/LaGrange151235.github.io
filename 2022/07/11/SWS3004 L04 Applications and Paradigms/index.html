SWS3004 L04 Applications and Paradigms

发表于 2022-07-11 更新于 2022-07-12 分类于 NUS Summer Workshop 2022

Common Features of Cloud Providers

  • Basic and higher-level services (IaaS -> SaaS): different level of abstraction provides flexibility while developing applications
  • Compute and storage resources: virtual servers (Linux and Windows) and object store
  • Deployment management services: load balancer, auto-scaling, message queueing, monitoring, …
  • User interface: graphical user interface, command-line interface

Cloud Applications

  • ideal applications: massive computing
    • web
    • database
    • search
    • ML with massive-scale models
  • unideal applications: complex workflow, multiple dependencies, intensive communication and arbitrarily inpartible

Challenges in Developing Cloud Applications

  • Performance isolation
  • Reliability
  • Latency and bandwidth fluctuations
  • Performance limit

Architectural Styles for Cloud Applications

  • “as-a-service” cloud delivery models
    • Simple Object Access Protocol(SOAP)
      • based on then XML
      • uses TCP or UDP transport protocols
    • Representational State Transfer(REST)
      • usually use JSON

Web Services for Cloud Applications

  • Cloud services are naturally distributed
  • Cloud services are accessed by different clients using different programming languages and technologies

Infrastructure as a Service

  • Web access to resources
  • Centralized physical resource management
  • Elastic services and dynamic scaling
  • Shared infrastructure across multiple users
  • Preconfigured VMs
  • Metered services

Platform as a Service

  • All in one – same IDE to develop, test, deploy, host and maintain applications
  • Web access to development platforms
  • Offline access for developers
  • Built-in scalability
  • Collaborative platform for developers
  • Diverse client tools

Software as a Service

  • Multi-tenanted applications
  • Web access
  • Centralized management of SaaS services
  • Multi-device(and multi-browser) support
  • Scalability under varying loads
  • High availability
  • API integration with other software

Examples of Setting up a Blog

SaaS

  1. Accesses a server with Wordpress installed and
    configured
  2. Setup a new blog
  3. Blogs happily about SaaS development

PaaS

  1. Accesses a system with Linux, Apache, MySQL and PHP
    (LAMP)
  2. Installs and configures Wordpress blog engine
  3. Setup a new blog
  4. Blogs happily about PaaS provisioning

IaaS

  1. Creates a compute instance running Linux
  2. Installs and configures Apache, MySQL and PHP
  3. Installs and configures Wordpress blog engine
  4. Configures a new blog
  5. Blogs happily about IaaS development

*FaaS

  • Event-driven system
  • use lambda functions triggered by events
  • implement severless applications without need to provision or manage servers
