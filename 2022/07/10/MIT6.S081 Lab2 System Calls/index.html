System call tracing

  • Hardness: moderate
  • Dtails: 用一个mask值来按位设置是否跟踪对应的系统调用，对于跟踪的系统调用，输出格式为
    $$pid:syscall\quad{syscall_{name}}\rightarrow{return_{value}}$$
  • Implementaion:
    1. 设置系统调用编号
    2. 声明系统调用函数，生成映射
    3. 声明跳板函数
    4. 声明用户态函数
    5. 实现内核态功能
      首先在proc结构体中对每一个进程添加一个记录mask的变量int tracingMask，二进制形式中tracingMask第n位为1则对第n个syscall进行跟踪，否则不跟踪 然后在fork()函数中令生成的子进程复制父进程的tracingMask值，实现对子进程的跟踪 然后在内核态实现从命令中获取mask值并赋给此时调用进程的tracingMask的功能 最后在syscall()函数中添加关于tracingMask的判断和输出，这样每当被跟踪的syscall被调用时，就可以输出需要的信息

Sysinfo

  • Hardness: moderate
  • Dtails: 对整个系统资源进行统计，填充结构体struct sysinfo，由测试程序sysinfotest来测试系统调用sysinfo的功能，struct sysinfo包含两个变量freemem和nproc，分别记录系统中空闲内存的字节数和正在运行的进程的数量
  • Implementation:
    1. 设置系统调用编号
    2. 声明系统调用函数，生成映射
    3. 声明跳板函数
    4. 声明用户态函数
    5. 实现内核态功能
      首先实现对空闲内存的统计，空闲内存在内核中用一个链表结构来实现，通过阅读kalloc.c中原有的代码找到空闲页链表的表头freelist，遍历freelist就可以统计完所有的空闲内存，每一页都是4096byte 然后实现对正在运行的进程的数目，这里是通过遍历进程记录数组proc[NPROC]，在每个元素中通过state来确定是否正在运行，遍历一次就可以统计完毕 最后在syscall()函数中添加从内核态将信息复制到用户态的函数实现，根据提示，从内核态复制信息到用户态需要copyout()函数
