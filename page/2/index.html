System call tracing

  • Hardness: moderate
  • Dtails: 用一个mask值来按位设置是否跟踪对应的系统调用，对于跟踪的系统调用，输出格式为
    $$pid:syscall\quad{syscall_{name}}\rightarrow{return_{value}}$$
  • Implementaion:
    1. 设置系统调用编号
    2. 声明系统调用函数，生成映射
    3. 声明跳板函数
    4. 声明用户态函数
    5. 实现内核态功能
      首先在proc结构体中对每一个进程添加一个记录mask的变量int tracingMask，二进制形式中tracingMask第n位为1则对第n个syscall进行跟踪，否则不跟踪 然后在fork()函数中令生成的子进程复制父进程的tracingMask值，实现对子进程的跟踪 然后在内核态实现从命令中获取mask值并赋给此时调用进程的tracingMask的功能 最后在syscall()函数中添加关于tracingMask的判断和输出，这样每当被跟踪的syscall被调用时，就可以输出需要的信息

Sysinfo

  • Hardness: moderate
  • Dtails: 对整个系统资源进行统计，填充结构体struct sysinfo，由测试程序sysinfotest来测试系统调用sysinfo的功能，struct sysinfo包含两个变量freemem和nproc，分别记录系统中空闲内存的字节数和正在运行的进程的数量
  • Implementation:
    1. 设置系统调用编号
    2. 声明系统调用函数，生成映射
    3. 声明跳板函数
    4. 声明用户态函数
    5. 实现内核态功能
      首先实现对空闲内存的统计，空闲内存在内核中用一个链表结构来实现，通过阅读kalloc.c中原有的代码找到空闲页链表的表头freelist，遍历freelist就可以统计完所有的空闲内存，每一页都是4096byte 然后实现对正在运行的进程的数目，这里是通过遍历进程记录数组proc[NPROC]，在每个元素中通过state来确定是否正在运行，遍历一次就可以统计完毕 最后在syscall()函数中添加从内核态将信息复制到用户态的函数实现，根据提示，从内核态复制信息到用户态需要copyout()函数

sleep

  • Hardness: Easy
  • Details: 实现休眠指令
  • Implementation: 这里主要需要的技巧是C语言main函数在终端中的接口，
    • argc是参数的数目，这里的数目是包含命令本身的，也就是说对于一个命令
      $$command\quad$$
      argc的值应该是$1+sizeof(parameters)$
    • argv是参数的列表，由上条就可以知道我们应当从argv[1]开始读取${parameters}$列表

pingpong

  • Hardness: Easy
  • Details: 用管道pipe实现一个类似与pingpong的双向管道通信程序
  • Implementation: 这里主要需要的技巧是使用xv6中建立管道pipe并利用它们进行通信
    • pipe()函数可以初始化一个一维数组作为管道pipe
    • read()函数用于从管道读取信息
      • 参数1是pipe的读文件描述符，一般是pipe[0]
      • 参数2是读buffer的地址
      • 参数3是读取的长度
    • write()函数用于从管道写入信息
      • 参数1是pipe的写文件描述符，一般是pipe[1]
      • 参数2是写buffer的地址
      • 参数3是写入的长度

  • Details: 具体内容参见给出链接的论文Bell Labs and CSP Threads，通过传递的管道通信来求出35以内的素数，部分伪代码如下
  • Implementation: 这里主要需要考虑的问题是在xv6中，每个进程只能支持16个文件描述符，因此我们总是需要使用close()及时关闭不需要的文件描述符

find

  • Hardness: moderate
  • Details: 实现文件查找，这里的查找需要支持文件夹之下内容的查找
  • Implementation: 这里主要是通过递归的调用find在给出的path中搜索target文件，搜索比对的方法是在path的字符串中切割出长度为target文件名长度的一部分进行比较，需要注意的是要跳过文件符’.’和’..’，他们分别指代当前文件夹路径和上级文件夹路径

xargs

  • Hardness: moderate
  • Details: xargs是Unix和类Unix系统中的一个重要指令，使用xargs的指令一般形式如下：
    $${commands}|xargs\quad{command}$$
    这里的|符号是管道pipe操作符，意思是将|左侧的指令执行的结果作为参数通过管道传递给管道右侧的指令，实现一个指令对多个参数的执行
  • Implementation: 这里主要是通过封装好的run()函数，通过fork()建立新的进程，分别对每一个读取到的参数执行命令
